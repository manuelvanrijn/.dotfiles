# Global coding standards

- Be extremely concise. Sacrifice grammar for brevity.
- Before replying, use relevant skills/tools proactively when they apply.
- Simplicity first: make the smallest change that solves the problem.
- Find the root cause. No temporary fixes.
- Think holistically to find the right scope, then minimize the change within it.
- Assume unreleased branch work can be changed directly.
- Avoid compatibility shims unless explicitly needed.
- If part of a feature is not wired yet, stub it safely without breaking UX.
- Prefer clear names over abbreviations or single letters; use `custom_field`, not `cf` or `c`.
